ansible all -i 172.22.55.20, -e ansible_user=ec2-user -e ansible_password=DevOps321 -m ping
ansible all -i 172.22.55.20, -e ansible_user=ec2-user -e ansible_password=DevOps321 -m package -b -a "name=nginx state=present"
ansible all -i 172.22.55.20, -e ansible_user=ec2-user -e ansible_password=DevOps321 -m service -b -a "name=nginx state=started"

-i (Inventory): Specifies the target host or inventory file.
The trailing comma (172.22.55.20,) trick forces Ansible to treat the string as a direct list of IP addresses instead of looking for a file.
-e (Extra Variables): Passes runtime variables to the command. Here, it explicitly sets the SSH username (ansible_user=ec2-user) and the login password (ansible_password=DevOps321) for the target machine.
-m (Module): Defines the Ansible module to execute. The package module is an OS-agnostic manager that automatically calls yum, apt, or dnf depending on the target system.
-a (Arguments): Passes the required arguments to the chosen module. In this case, it instructs the package manager to ensure nginx is installed (state=present).
-b become root
